Welcome to Darwin/ARM

Welcome to Darwin/ARM. Darwin is the core operating system that lies under both iPhone OS and Mac OS X. It mainly consists of the kernel and various other core components such as dyld, launchd and such. Over time, Apple has not really paid much attention to their open-source releases as present on the open source portal, including not publishing code for the ARM version of the xnu kernel.

This project aims to fill that gap.

Current Status

Darwin/ARM runs on a variety of ARM reference platforms and hardware such as:

  • ARM RealView (Platform Baseboard for Cortex-A8/Emulation Baseboard)
  • Texas Instruments DM3730 (BeagleBoard xM)
  • Texas Instruments Sitara AM335x (BeagleBone)
  • Samsung S5L89xx (All ARMv7 platforms from 8920 to 8930).

Currently, the kernel can perform a variety of tasks, including running unmodified binaries from iPhone OS.

Userspace

Currently, the userspace is very simple, with the FreeBSD init, a simple runcom script and a getty. Multiuser does work at the moment. Many CoreFoundation/IOKit apps run at the moment, such as sw_vers, or ioreg.

The OS is incredibly volatile, so be warned! (There are dragons lurking around.)

Building

Building is relatively simple if you're on OS X, simply compile ctfmerge, ctfconvert and ctfdump from the dtrace package. Then check out the kernel source and simply build for your platform using `make TARGET_CONFIGS="debug arm ".

A sample boot loader is also available here for ARM RealView PB-A8 and OMAP3.

General purpose kernelcache build support coming soon! (Kernelcaches can currently be used, but building is rather unorthodox at the moment)
